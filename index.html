Mission 01

Airline Ticketing
Cloud Migration

Microsoft Azure Python Bash Oracle SQL Jenkins Grafana
Context

Progressive migration of a multi-airline ticketing platform from on-premise to Microsoft Azure — requiring strong observability, and test reliability.

Challenges
  • Log access tightly coupled to on-premise infrastructure — non-regression (NR) tests started failing after migration to Azure
  • Critical production performance degradation: SQL queries went from milliseconds to minutes after Azure migration
  • 50 identical Jenkinsfiles duplicated across repositories — unmaintainable at scale
  • NR tests failed due to heterogeneous airline environments (on-prem vs Azure, object storage vs persistent volumes) during migration.
What I did
01 · Automation
Dynamic log access
Python & Bash scripts auto-updating log-access logic across all repos for Azure instead of on-premise systems without manual changes.
02 · Performance
SQL investigation & fix
Traced Oracle suboptimal index selection. Identified root cause and benchmarked two fix strategies across 44 airlines.
03 · CI/CD
Jenkins centralization
50 duplicated Jenkinsfiles replaced by one centralized repo with a shared parameterized pipeline.
04 · Reliability
Adaptive NR tests
Implemented centralized Jenkins configuration with per-airline parameters and refactored test scripts to dynamically adapt to environment and storage backend at runtime.
Results
+27pp
NR test pass rate recovered — 62% → 89% after Azure migration
~100×
SQL query speedup — from several minutes back to milliseconds
50 → 1
Jenkinsfiles centralized — one parameterized pipeline for all repos
Seamless Migration
Restored reliable NR test execution across all environments and enabled seamless migration with mixed configurations.
Mission 02

Order Management
System

Argo CD Kubernetes Helm Kafka / Strimzi Solace Event Mesh MongoDB Jenkins OpenShift
Context

Order Management System designed to handle airline orders in a high-availability environment. The platform is built on microservices deployed on Kubernetes, leveraging Event-Driven and Service-Oriented Architectures. It operates across multiple environments (INT, Development, UAT, Pre-Production, and Production), distributed over several clusters.

Challenges
  • OOM crashes caused by Kafka consumer lag — intermittent instability
  • CI/CD pod reuse causing memory leaks; broken PR promotion and build-status logic
  • Faulty orders blocking entire Kafka partitions, stalling downstream processing
  • Manual OpenShift changes on customer-facing environments bypassing CI/CD pipelines undetected
  • MongoDB CPU saturation (100%) during migration, impacting all dependent microservices
  • Multi-cluster Kafka migration with strict 5-minute downtime constraint and zero data loss tolerance
What I did
01 · Incidents
Production L2/L3 response
Resolved recurring OOM caused by Kafka lag via memory limit tuning. Built a controlled skip mechanism for partition-blocking orders. Deep-traced a Splunk missing-index failure causing endless EDA retries, cross-reference issues between a European airline and a Gulf carrier airline systems, and incorrect HTTP paths on deployed microservices.
02 · CI/CD
Pipeline reliability fixes
Fixed pod reuse memory leaks by enforcing fresh agent creation per build phase. Added per-package Helm retry logic against transient network failures. Fixed PR build-status logic to always consider last result. Automated PR job creation before scan to eliminate false-success merges.
03 · Automation
CI/CD promotion & Teams alerts
Designed a single-PR workflow — developer submits one PR for the first environment only. Pipeline auto-promotes across environments, creates PRs per phase, and posts real-time build status (creation, success, failure, merge) into the exact Teams thread via Power Automate.
04 · Governance
Audit log drift detection
Detected manual env variable, feature toggle, and version changes on customer-facing OpenShift systems. Collected K8s audit logs into a scoped Splunk index and triggered Teams alerts with who/what/when for every change — enabling immediate revert and forcing justification for critical modifications.
05 · Migration
Kafka & MongoDB migrations
Gulf carrier airline UAT → PreProd: Kafka topic replication, custom Bash offset migration script (beyond MirrorMaker limits), Solace Event Mesh queue realignment, ACL fix and Kafka Topic Operator fix via Argo CD, K8s namespace deployment. European airline MongoDB: resolved 100% CPU saturation by reducing simultaneous collection injections and adding time delays between document batches.
06 · Evaluation
Solace Event Mesh self-service stabilization
Tested the new Solace Event Mesh self-service microservice in Kubernetes for production readiness. Identified replica count desync with Custom Resource, duplicate Kafka Topic Operator object generation, and config file bloat impacting readability. Drove technical reviews with the Solace team, leading to significant stability improvements.
